Структура маршрутов
Приложение содержит следующие маршруты :
+1

/ — Главная страница (Home).

/about — О программе (About).

/courses — Список курсов с поддержкой сортировки через query-параметры.

/courses/:id — Детальная страница курса (динамический маршрут).

* — Страница 404 для несуществующих адресов.

Технический разбор: useParams vs useSearchParams
В этой лабораторной работе я использовал два разных подхода для работы с URL, как описано в учебнике:
+4

1. useParams (URL Parameters) 
+2

Где использовал: В компоненте CourseDetail.tsx для получения :id.

Когда использовать: Используется для обязательных данных, без которых страница не может существовать. Например, конкретный ID ресурса в пути /courses/1.


Синтаксис: Переменная объявляется в роутере через двоеточие (:id).

2. useSearchParams (Query Parameters) 
+3

Где использовал: В компоненте Courses.tsx для сортировки списка (?sort=asc).

Когда использовать: Используется для необязательных параметров, которые меняют способ отображения данных на странице (фильтры, сортировка, поиск), но не саму суть страницы.


Синтаксис: Работает как состояние (state), позволяя читать и изменять параметры строки запроса без перезагрузки страницы.
+1
